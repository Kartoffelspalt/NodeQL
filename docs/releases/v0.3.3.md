# NodeQL 0.3.3

NodeQL 0.3.3 improves project reliability and makes the visual SQL workspace
more readable.

## Project files

- New projects create a `.nodeql` file immediately and choose a unique name if
  a file with the selected name already exists.
- `Cmd+S` on macOS and `Ctrl+S` on Windows and Linux save the active project.
- Autosave can be enabled per project when it is created.
- Older supported workspace, NodeQL, and ScratchQL project files show an
  upgrade dialog when opened.
- Before an upgrade, NodeQL creates a timestamped backup next to the original
  project file.
- Files created by a newer incompatible NodeQL version are not overwritten.

## Workspace

- Node labels and input slots are positioned more consistently.
- Wider workspace nodes and palette previews prevent long labels and values
  from being clipped.
- JOIN nodes retain their specialized two-row layout.

## Platform and documentation

- macOS no longer uses obsolete CocoaPods integration for the current Swift
  package plugin setup.
- The NodeQL architecture wiki is now available in English.
- `build_runner` was updated to `2.15.1`.
